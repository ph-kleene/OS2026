# OS 作业 2 答案

## 第 1 题

题目要求：用 `fork`、`exit`、`wait` 实现一个采用随机算法计算 π 值的多进程程序。子进程负责随机试验并通过 `exit` 将结果返回给父进程，父进程负责汇总并输出最终 π 值（设置适当的子进程数量，使得计算结果的小数点后两位多次运行结果稳定）。

### 1.1 设计思路

- **随机算法**：蒙特卡洛方法。在 $[0,1] \times [0,1]$ 单位正方形内随机投点，统计落在四分之一圆 ($x^2 + y^2 \le 1$) 内的比例，$\pi \approx 4 \times \frac{\text{命中数}}{\text{总投点数}}$。
- **进程分工**：父进程 fork 出 1000 个子进程。每个子进程独立执行 255 次投点试验，将命中次数通过 `exit(hits)` 返回（255 ≤ 8-bit 上限，安全）。父进程用 `wait()` 逐一回收子进程，用 `WEXITSTATUS` 提取命中数，汇总后计算 π。
- **参数选择**：1000 子进程 × 255 次/子进程 = 255,000 总试验点。蒙特卡洛误差 $\approx 1/\sqrt{N} \approx 0.002$，小数点后两位稳定在 3.14。
- **缓冲区处理**：`fork()` 前 `fflush(stdout)`，避免子进程继承父进程 stdio 缓冲区导致重复输出。
- **随机数**：子进程用 `srand48(time(NULL) ^ (getpid() << 16))` 生成独立种子，调用 `drand48()` 做 48-bit 精度随机采样。

### 1.2 源码 `multi_process_pi.c`

```c
#define _GNU_SOURCE
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/wait.h>
#include <time.h>
#include <math.h>

#define NUM_CHILDREN      1000
#define TRIALS_PER_CHILD  255

int main(void)
{
    int total_hits    = 0;
    int total_trials  = NUM_CHILDREN * TRIALS_PER_CHILD;
    int completed     = 0;

    printf("=== Monte Carlo Pi (Multi-Process) ===\n");
    printf("Children: %d, Trials per child: %d\n", NUM_CHILDREN, TRIALS_PER_CHILD);
    printf("Total trials: %d\n\n", total_trials);
    fflush(stdout);  /* prevent children inheriting buffered output */

    for (int i = 0; i < NUM_CHILDREN; i++) {
        pid_t pid = fork();
        if (pid < 0) {
            perror("fork");
            exit(EXIT_FAILURE);
        }
        if (pid == 0) {
            /* Child: run random trials and return hit count via exit() */
            srand48((long)(time(NULL) ^ (getpid() << 16)));

            int hits = 0;
            for (int j = 0; j < TRIALS_PER_CHILD; j++) {
                double x = drand48();
                double y = drand48();
                if (x * x + y * y <= 1.0)
                    hits++;
            }
            exit(hits); /* hit count fits in 8-bit exit code (max 255) */
        }
    }

    /* Parent: wait for all children and aggregate results */
    while (completed < NUM_CHILDREN) {
        int status;
        pid_t pid = wait(&status);
        if (pid < 0) {
            perror("wait");
            continue;
        }
        if (WIFEXITED(status)) {
            total_hits += WEXITSTATUS(status);
            completed++;
        }
    }

    double pi    = 4.0 * total_hits / total_trials;
    double error = fabs(pi - M_PI);

    printf("Estimated pi = %.6f\n", pi);
    printf("Actual pi    = %.6f\n", M_PI);
    printf("Error        = %.6f\n", error);
    printf("Hits: %d / %d\n", total_hits, total_trials);

    return 0;
}
```

### 1.4 关键 API 使用说明

| API | 使用位置 | 作用 |
|-----|---------|------|
| `fork()` | 父进程循环中 | 创建子进程，返回 0 表示子进程，>0 表示父进程 |
| `exit(hits)` | 子进程 | 将命中次数作为退出状态返回给父进程（低 8 位） |
| `wait(&status)` | 父进程循环中 | 阻塞等待任意子进程结束，回收僵尸进程 |
| `WIFEXITED(status)` | 父进程 | 判断子进程是否正常退出 |
| `WEXITSTATUS(status)` | 父进程 | 提取子进程的 8 位退出码（命中次数） |

## 第 2 题

题目要求：使用 `signal`、`ualarm`、`raise` 和 `ucontext` 系列 API 实现一个简易的用户态多线程库（时间片轮转法，时间片 20ms），提供 `thread_create`、`thread_join`、`thread_yield`、`thread_destroy` 等 API。基于该线程库实现多线程蒙特卡洛求 π 程序。

### 2.1 线程库设计

#### 2.1.1 整体架构

```
┌──────────────────────────────────────────────────────┐
│                    User Process                       │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐              │
│  │ Thread 0 │  │ Thread 1 │  │ Thread N │   ...       │
│  │ (main)   │  │          │  │          │              │
│  └────┬─────┘  └────┬─────┘  └────┬─────┘              │
│       │              │              │                   │
│       └──────────────┼──────────────┘                   │
│                      │                                  │
│              ┌───────▼────────┐                        │
│              │   Scheduler    │                        │
│              │ (sched_loop)   │                        │
│              └───────┬────────┘                        │
│                      │                                  │
│         ┌────────────┼────────────┐                    │
│         │            │            │                     │
│    SIGALRM      raise()     swapcontext                │
│   (20ms timer)  (yield)    (voluntary)                 │
└──────────────────────────────────────────────────────┘
```

- 每个线程拥有独立的 `ucontext_t` 上下文和 `malloc` 分配的 128KB 栈。
- 调度器自身也拥有独立的上下文和栈，执行 `sched_loop()` 调度循环。
- 主线程（`main()`）被包装为 `tid=0` 的线程参与调度。

#### 2.1.2 时间片轮转调度

**抢占路径**（`SIGALRM` 定时器）：

1. `ualarm(20000, 20000)` 每 20ms 产生 `SIGALRM` 信号。
2. 信号处理函数通过 `sigaction` + `SA_SIGINFO` 获取第三个参数 `uap`——这是内核保存的**被中断线程的 `ucontext_t`**。
3. 将被中断线程的上下文原样拷贝到 TCB 的 `ctx` 字段，标记为 `READY`，重新入队。
4. 调用 `setcontext(&sched_ctx)` 跳转到调度器（`sched_loop`），丢弃信号处理函数自身的上下文。

**自愿让权路径**（`thread_yield`）：

1. 线程调用 `raise(SIGALRM)`，同步触发同一个信号处理函数。
2. 流程与抢占路径相同：保存上下文 → 入队 → 进调度器。

**调度器逻辑**（`sched_loop`）：

```c
while (1) {
    next = dequeue();          // 从就绪队列取下一个线程
    current = next;
    current->state = TS_RUNNING;
    sigdelset(&current->ctx.uc_sigmask, SIGALRM);  // 确保定时器不被屏蔽
    swapcontext(&sched_ctx, &current->ctx);        // 切换上下文

    // 从 swapcontext 返回 = 线程被抢占 / yield / 结束
    if (current->state == TS_FINISHED) {
        current = NULL;        // wrapper 已唤醒 joiner
    }
}
```

#### 2.1.3 线程生命周期

```
                 thread_create()
                       │
                       ▼
  ┌──────┐        ┌─────────┐    swapcontext     ┌──────────┐
  │ INIT │ ──────>│  READY  │ ───────────────>   │ RUNNING  │
  └──────┘        └─────────┘                    └────┬─────┘
                       ▲                             │
                       │          preempt / yield    │
                       └─────────────────────────────┘
                                                     │
                                               func returns
                                                     │
                                                     ▼
                                              ┌──────────┐   thread_destroy
                                              │ FINISHED │ ───────────────>
                                              └──────────┘     (free TCB)

  RUNNING ── thread_join(tid) ──> BLOCKED ── target finishes ──> READY
```

#### 2.1.4 API 说明

| API | 功能 |
|-----|------|
| `thread_create(func, arg)` | 分配 128KB 栈 + TCB，用 `getcontext` + `makecontext` 设置入口为 `thread_wrapper`，加入就绪队列，返回 tid |
| `thread_join(tid)` | 若目标已结束则立即返回；否则标记当前线程为 `BLOCKED`，在目标 TCB 中记录 joiner，`swapcontext` 切换到调度器 |
| `thread_yield()` | 调用 `raise(SIGALRM)`，复用信号处理函数完成上下文保存和调度切换 |
| `thread_destroy(tid)` | 释放已 `FINISHED` 线程的栈和 TCB 内存（不可销毁主线程 tid=0） |
| `thread_self()` | 返回当前运行线程的 tid |

### 2.2 源码

#### 2.2.1 头文件 `user_thread.h`

```c
#ifndef USER_THREAD_H
#define USER_THREAD_H

#define THREAD_STACK_SIZE   (128 * 1024)
#define SCHED_STACK_SIZE    (128 * 1024)
#define MAX_THREADS         128
#define TIME_SLICE_US       20000         /* 20 ms */

int  thread_create(void (*func)(void *), void *arg);
int  thread_join(int tid);
void thread_yield(void);
void thread_destroy(int tid);
int  thread_self(void);
```

#### 2.2.2 线程库核心 `user_thread.c`

```c
#define _GNU_SOURCE
#include "user_thread.h"

#include <signal.h>
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <time.h>
#include <ucontext.h>
#include <unistd.h>

/* ----- thread states ----- */
typedef enum {
    TS_READY,
    TS_RUNNING,
    TS_BLOCKED,
    TS_FINISHED
} thread_state_t;

/* ----- thread control block ----- */
typedef struct thread {
    int             tid;
    thread_state_t  state;
    ucontext_t      ctx;
    void           *stack;
    void          (*func)(void *);
    void           *arg;
    struct thread  *joiner;
    struct thread  *next;
} thread_t;

/* ----- global scheduler state ----- */
static thread_t  *current        = NULL;
static thread_t  *ready_head     = NULL;
static thread_t  *ready_tail     = NULL;
static thread_t  *main_th        = NULL;
static ucontext_t sched_ctx;
static char       sched_stack[SCHED_STACK_SIZE];
static int        next_tid       = 1;
static int        init_done      = 0;
static thread_t  *all_threads[MAX_THREADS];
static int        num_threads    = 0;

/* ----- helpers ----- */
static void enqueue(thread_t *t)
{
    t->next = NULL;
    if (!ready_head) {
        ready_head = ready_tail = t;
    } else {
        ready_tail->next = t;
        ready_tail = t;
    }
}

static thread_t *dequeue(void)
{
    if (!ready_head) return NULL;
    thread_t *t = ready_head;
    ready_head = t->next;
    if (!ready_head) ready_tail = NULL;
    t->next = NULL;
    return t;
}

static thread_t *find_thread(int tid)
{
    for (int i = 0; i < num_threads; i++)
        if (all_threads[i]->tid == tid)
            return all_threads[i];
    return NULL;
}

static void thread_wrapper(thread_t *t);
static void sched_loop(void);

/* ----- SIGALRM handler -----
 *
 * Uses SA_SIGINFO so the third argument (uap) supplies the ucontext_t of the
 * *interrupted* code.  We copy that into the current thread's ctx, mark the
 * thread ready, and transfer control to the scheduler with setcontext().
 *
 * The handler never returns — setcontext() does not save the handler's own
 * context, which is fine because the preempted state is already preserved.
 */
static void sigalrm_handler(int sig, siginfo_t *info, void *uap)
{
    (void)sig;
    (void)info;

    ualarm(TIME_SLICE_US, TIME_SLICE_US);

    if (!current || current->state != TS_RUNNING)
        return;

    current->ctx = *(ucontext_t *)uap;
    current->state = TS_READY;
    enqueue(current);

    setcontext(&sched_ctx);
}

/* ----- scheduler loop -----
 *
 * Round-robin: dequeue next ready thread, mark it RUNNING, swap to it.
 */
static void sched_loop(void)
{
    while (1) {
        thread_t *next = dequeue();
        if (!next) {
            if (main_th && main_th->state == TS_BLOCKED) {
                struct timespec ts = { .tv_sec = 0, .tv_nsec = 1000000 };
                nanosleep(&ts, NULL);
                continue;
            }
            break;
        }

        current = next;
        current->state = TS_RUNNING;

        sigdelset(&current->ctx.uc_sigmask, SIGALRM);

        swapcontext(&sched_ctx, &current->ctx);

        if (current->state == TS_FINISHED) {
            current = NULL;
        }
    }
}

/* ----- thread wrapper -----
 *
 * Runs on the new thread's own stack.  Calls the user function, then marks
 * the thread FINISHED, wakes the joiner, and swaps to the scheduler forever.
 */
static void thread_wrapper(thread_t *t)
{
    t->func(t->arg);

    t->state = TS_FINISHED;

    if (t->joiner) {
        t->joiner->state = TS_READY;
        enqueue(t->joiner);
    }

    swapcontext(&t->ctx, &sched_ctx);
}

/* ----- public API ----- */

void thread_scheduler_init(void)
{
    if (init_done) return;

    main_th = calloc(1, sizeof(thread_t));
    if (!main_th) { perror("calloc"); exit(EXIT_FAILURE); }
    main_th->tid   = 0;
    main_th->state = TS_RUNNING;
    main_th->stack = NULL;
    current = main_th;

    all_threads[num_threads++] = main_th;

    if (getcontext(&sched_ctx) == -1) { perror("getcontext"); exit(EXIT_FAILURE); }
    sched_ctx.uc_stack.ss_sp    = sched_stack;
    sched_ctx.uc_stack.ss_size  = sizeof(sched_stack);
    sched_ctx.uc_link           = NULL;
    makecontext(&sched_ctx, sched_loop, 0);

    struct sigaction sa;
    memset(&sa, 0, sizeof(sa));
    sa.sa_sigaction = sigalrm_handler;
    sigemptyset(&sa.sa_mask);
    sa.sa_flags = SA_SIGINFO | SA_RESTART;
    if (sigaction(SIGALRM, &sa, NULL) == -1) { perror("sigaction"); exit(EXIT_FAILURE); }

    ualarm(TIME_SLICE_US, TIME_SLICE_US);

    init_done = 1;
}

int thread_create(void (*func)(void *), void *arg)
{
    if (!init_done || !func || num_threads >= MAX_THREADS) return -1;

    thread_t *t = calloc(1, sizeof(thread_t));
    if (!t) return -1;
    t->stack = malloc(THREAD_STACK_SIZE);
    if (!t->stack) { free(t); return -1; }

    t->tid   = next_tid++;
    t->state = TS_READY;
    t->func  = func;
    t->arg   = arg;

    if (getcontext(&t->ctx) == -1) { free(t->stack); free(t); return -1; }
    t->ctx.uc_stack.ss_sp   = t->stack;
    t->ctx.uc_stack.ss_size = THREAD_STACK_SIZE;
    t->ctx.uc_link          = NULL;
    makecontext(&t->ctx, (void (*)(void))thread_wrapper, 1, t);

    sigdelset(&t->ctx.uc_sigmask, SIGALRM);

    enqueue(t);
    all_threads[num_threads++] = t;

    return t->tid;
}

int thread_join(int tid)
{
    if (!init_done || tid == current->tid) return -1;

    thread_t *target = find_thread(tid);
    if (!target) return -1;

    if (target->state == TS_FINISHED)
        return 0;

    current->state  = TS_BLOCKED;
    target->joiner  = current;

    swapcontext(&current->ctx, &sched_ctx);

    return 0;
}

void thread_yield(void)
{
    if (!init_done) return;
    raise(SIGALRM);
}

void thread_destroy(int tid)
{
    if (!init_done || tid == 0) return;

    thread_t *target = find_thread(tid);
    if (!target || target->state != TS_FINISHED) return;

    for (int i = 0; i < num_threads; i++) {
        if (all_threads[i] == target) {
            all_threads[i] = all_threads[--num_threads];
            break;
        }
    }

    free(target->stack);
    free(target);
}

int thread_self(void)
{
    if (!init_done || !current) return -1;
    return current->tid;
}
```

#### 2.2.3 多线程 π 程序 `multi_thread_pi.c`

```c
#define _GNU_SOURCE
#include "user_thread.h"

#include <stdio.h>
#include <stdlib.h>
#include <sys/time.h>
#include <math.h>

#define NUM_THREADS         20
#define TRIALS_PER_THREAD   50000

typedef struct {
    int           id;
    int           trials;
    int          *results;
    unsigned int  seed;
} worker_arg_t;

static void worker(void *raw)
{
    worker_arg_t *a = (worker_arg_t *)raw;
    unsigned int  seed = a->seed;

    int hits = 0;
    for (int i = 0; i < a->trials; i++) {
        double x = (double)rand_r(&seed) / RAND_MAX;
        double y = (double)rand_r(&seed) / RAND_MAX;
        if (x * x + y * y <= 1.0)
            hits++;
    }
    a->results[a->id] = hits;
}

int main(void)
{
    thread_scheduler_init();

    worker_arg_t args[NUM_THREADS];
    int          results[NUM_THREADS];
    int          tids[NUM_THREADS];
    int          total_trials = NUM_THREADS * TRIALS_PER_THREAD;

    printf("=== Monte Carlo Pi (User-Threads) ===\n");
    printf("Threads: %d, Trials per thread: %d\n", NUM_THREADS, TRIALS_PER_THREAD);
    printf("Total trials: %d\n\n", total_trials);

    struct timeval tv;
    gettimeofday(&tv, NULL);
    unsigned int base_seed = (unsigned int)(tv.tv_sec ^ (tv.tv_usec << 12));

    for (int i = 0; i < NUM_THREADS; i++) {
        args[i].id      = i;
        args[i].trials  = TRIALS_PER_THREAD;
        args[i].results = results;
        args[i].seed    = base_seed ^ (unsigned int)(i * 2654435761U);
        tids[i] = thread_create(worker, &args[i]);
        if (tids[i] < 0) {
            fprintf(stderr, "thread_create failed at %d\n", i);
            return 1;
        }
    }

    int total_hits = 0;
    for (int i = 0; i < NUM_THREADS; i++) {
        thread_join(tids[i]);
        total_hits += results[i];
        thread_destroy(tids[i]);
    }

    double pi    = 4.0 * total_hits / total_trials;
    double error = fabs(pi - M_PI);

    printf("Estimated pi = %.6f\n", pi);
    printf("Actual pi    = %.6f\n", M_PI);
    printf("Error        = %.6f\n", error);
    printf("Hits: %d / %d\n", total_hits, total_trials);

    return 0;
}
```

### 2.4 关键 API 使用说明

| API | 使用位置 | 作用 |
|-----|---------|------|
| `signal` / `sigaction` | `thread_scheduler_init` | 注册 `SIGALRM` 信号处理函数，`SA_SIGINFO` 标志获取被中断上下文 |
| `ualarm` | `thread_scheduler_init` + handler | 启动 20ms 周期性定时器，handler 中重新装填 |
| `raise` | `thread_yield` | 同步触发 `SIGALRM`，复用抢占路径完成上下文切换 |
| `getcontext` | `thread_create` / `scheduler_init` | 获取当前上下文快照，为 `makecontext` 准备基底 |
| `makecontext` | `thread_create` / `scheduler_init` | 将上下文入口点设置为 `thread_wrapper` / `sched_loop`，绑定栈 |
| `swapcontext` | `sched_loop` / `thread_wrapper` / `thread_join` | 保存当前上下文到第一参数，从第二参数恢复执行 |
| `setcontext` | `sigalrm_handler` | 直接跳转到调度器上下文（不保存 handler 上下文） |

### 2.5 `setcontext` vs `swapcontext` 的选择

- **信号处理函数**使用 `setcontext`：被中断线程的状态已通过 `uap` 拷贝保存，handler 自身无需恢复，直接跳转即可。
- **调度器**使用 `swapcontext`：调度器需要在每次切换线程后**保存自身状态**，以便下次线程让出 CPU 时能回到调度循环继续分发。
- **线程 wrapper**使用 `swapcontext`：线程结束后需要保存 wrapper 末尾的上下文（虽然不会再被唤醒），并切换到调度器。
