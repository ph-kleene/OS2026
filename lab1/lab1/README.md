## 项目结构

- **bootloader/**: 引导加载程序代码。
  - `start.asm`: 汇编入口，负责从实模式切换到保护模式。
  - `boot.c`: C 语言编写的引导逻辑，负责将应用程序从磁盘加载到内存并跳转。
  - `io.asm`: 包含磁盘读取等底层 I/O 操作。
- **app/**: 操作系统内核/应用程序代码。
  - `start.asm`: 应用程序汇编入口。
  - `main.c`: 主程序逻辑，通过读取 CMOS RTC 显示当前时间。
  - `io.asm`: 包含屏幕显示和 RTC 读取的底层 I/O 操作。
- **Makefile**: 主构建文件，负责编译引导程序、应用程序并生成磁盘镜像 `os.img`。

## 执行链路

1. BIOS 把磁盘第 0 扇区加载到物理地址 `0x7c00`，然后跳转到 bootloader 入口执行。
2. `bootloader/start.asm` 先在 16 位实模式下完成最基础的初始化，再切换到 32 位保护模式。
3. 切换完成后，`start.asm` 跳转到 `bootloader/boot.c` 中的 `bootMain()`。
4. `bootMain()` 调用 `bootloader/io.asm` 中的 `readSect()`，把磁盘上的 `app.bin` 从第 1 扇区开始读到物理地址 `0x10000`。
5. bootloader 最后跳转到 `0x10000`，开始执行 `app/start.asm` 和后续应用代码。

## 磁盘镜像布局 (`os.img`)

项目生成的 `os.img` 是一个标准的 1.44MB 软盘镜像，其布局如下：

| 偏移 (扇区) | 大小 | 内容 | 说明 |
| :--- | :--- | :--- | :--- |
| 0 | 512B | `bootloader.bin` | 主引导记录 (MBR)，包含 `0x55AA` 标志 |
| 1 - 128 | 64KB | `app.bin` | 应用程序模块 |
| 129 - 2879 | - | 空白 | 填充至 1.44MB |

## 内存布局

系统运行时的内存映射如下：

| 地址范围 | 大小 | 用途 |
| :--- | :--- | :--- |
| `0x00007C00 - 0x00007DFF` | 512B | 引导加载程序加载地址 |
| `0x00010000` 以下 | - | 系统与应用栈空间 (栈顶: `0x10000`) |
| `0x00010000 - 0x0001FFFF` | 64KB | 应用程序加载地址 |
| `0x000B8000 - 0x000B8FA0` | ~4KB | VGA 文本模式显存 (80x25) |

### 分段机制 (GDT)

系统采用了 32 位保护模式，GDT 配置如下：
- **粒度 (Granularity)**: 字节 (Byte)，G 位设为 0。
- **段长 (Limit)**: 最大为 1MB (`0xFFFFF` 字节)。
- **段基址 (Base)**:
  - 代码段/数据段: `0x00000000`
  - 显存段 (`gs`): `0x000B8000`

## 功能特性

1. **引导加载**: 自定义的引导程序能够将应用程序从软盘（镜像）的后续扇区加载到物理地址 `0x10000`。
2. **保护模式**: 引导程序完成了从 16 位实模式到 32 位保护模式的切换。
3. **RTC 时钟**: 应用程序通过读取 RTC 寄存器（BCD 码格式），并在屏幕中心实时刷新显示当前时间。


## 如何运行


### macOS m芯片的用户请注意
需要你们按照课程主页中m芯片的流程配置环境，同时注意修改`lab1/app/Makefile`以及`lab1/bootloader/Makefile`（详情见`Makefile`的注释）。

### 编译

在项目根目录下运行：
```bash
make
```
这将生成 `os.img` 磁盘镜像。

### 运行

使用 QEMU 启动：
```bash
make run
```

### 调试

使用 GDB 调试模式启动：
```bash
make debug
```

### 清理

移除生成的二进制文件和镜像：
```bash
make clean
```
