## 实验设计说明

项目架构可以划分为四个主要模块，并在不同的执行阶段和特权级下运行：**构建与部署**、**引导加载**、**内核核心**、**用户应用与库**。

### 1. 层次结构与特权级

| 层次 | 模块/组件 | 运行环境 | 特权级 (Ring) | 作用 |
| :--- | :--- | :--- | :--- | :--- |
| **应用层** | `app/` (应用程序) | 保护模式 | Ring 3 | 实现具体的用户功能（如 RTC 演示）。 |
| **库函数层** | `lib/` (myos.lib) | 保护模式 | Ring 3 | 封装系统调用，提供高层 API (如 `printf`, `now`, `sleep`)。 |
| **内核层** | `kernel/` (OS 内核) | 保护模式 | Ring 0 | 系统初始化、中断/异常处理、系统调用处理、驱动管理。 |
| **引导层** | `bootloader/` | 16位实模式/32位保护模式 | N/A (启动时) | 加载内核和应用程序到指定内存位置。 |

### 模块内存与磁盘布局

#### 磁盘布局 (Disk Layout)

项目生成的 `os.img` 磁盘镜像结构如下：

| 扇区范围 (Sector) | 内容 | 最大容量 | 备注 |
| :--- | :--- | :--- | :--- |
| 0 | Bootloader | 512 B | BIOS 加载入口 |
| 1 - 128 | Kernel (kernel.bin) | 64 KB | 内核映像 |
| 129 - 256 | App (app.bin) | 64 KB | 用户应用程序 |
| 257 - 2879 | Free Space | ~1.18 MB | 可用空闲区域 |

#### 内存布局 (Memory Layout)

本实验采用 x86 保护模式下的**纯分段映射**规划方案，通过设置不同的段基址实现逻辑隔离。

| 模块 (特权级) | 基址 | 限长 | 逻辑子项 | 虚拟地址 (VA) | 物理地址 (PA) | 说明 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **内核 (Ring 0)** | `0x00000000` | 1 MB | VGA 显存 | `0x000B8000` | `0x000B8000` | 文本模式映射区 |
| | | | 内核代码和数据 | `0x00010000` | `0x00010000` | 内核链接与加载位置 |
| | | | 内核栈顶 | `0x00010000` | `0x00010000` | 64KB 边界 (向下增长) |
| **应用 (Ring 3)** | `0x00020000` | 128 KB | 应用代码和数据 | `0x00000000` | `0x00020000` | 映射到物理 128KB 处 |
| | | | 应用栈顶 | `0x00020000` | `0x00040000` | 128KB 边界 (向下增长) |

### 2. 执行链路

*   **引导加载流程**：
    1.  **BIOS 加载**：BIOS 将磁盘扇区 0 的 `bootloader.bin` 加载到内存并跳转执行。
    2.  **模式切换**：Bootloader 从 16 位实模式切换到 32 位保护模式。
    3.  **内核加载**：Bootloader 从磁盘读取 `kernel.bin` 到物理地址 `0x10000` 处。
    4.  **内核跳转**：Bootloader 跳转到内核入口 `kernel/start.asm`。
*   **内核初始化**：内核主函数 (`kernel.c`) 负责初始化所有关键硬件和保护机制，包括 `initSerial` (串口)、`initIdt` (IDT)、`initIntr` (PIC)、`initTimer` (时钟) 和 `initSeg` (GDT)。
*   **应用加载与启动**：内核通过 `loadUMain()` 读取 `app.bin` 到物理地址 `0x20000` 处，并通过 IRET 指令将 CPU 切换到 Ring 3 执行应用代码。

### 3. 内存保护与用户态隔离

*   **分段隔离**：通过 GDT 定义了 Ring 0 (内核) 和 Ring 3 (应用) 的段描述符，实现特权级隔离。应用段的 Base Address 设为 `0x20000`，使应用代码在 VA 0 运行，但物理上与内核分离。
*   **中断与异常**：通过 `trap_entry.asm` 统一捕获所有中断、异常和系统调用，并由 `irq_dispatch.c` 进行分发处理。

### 4. 系统调用机制与用户库 (API)

*   **系统调用机制**：通过 `int 0x80` 软中断实现从用户态 (Ring 3) 到内核态 (Ring 0) 的特权级切换。
*   **内核服务**：内核提供了如 `SYS_WRITE` (打印), `SYS_NOW` (获取时间), `SYS_GET_TIME_TICKS` (休眠支持) 等系统调用。
*   **用户库封装**：`myos.lib` 将底层系统调用封装为高级 C 接口，包括精简版 `printf`、实时时钟获取 `now` 和基于时钟中断的延时 `sleep`。

### 5. 设备驱动支持

内核提供以下驱动支持：
*   **VGA/Serial 驱动**：支持文本模式下的显存操作和串口调试输出。
*   **Disk 驱动**：支持 LBA 模式进行扇区读取，用于加载模块。
*   **RTC 驱动**：从 CMOS 读取实时时钟 (Real Time Clock) 数据。
*   **Timer 驱动**：提供系统节拍 (System Tick)，支持时间管理和用户态 `sleep`。

## 构建与运行

### 1. 编译项目
在项目根目录下运行：
```bash
make os.img
```
该命令会依次编译引导程序、内核、用户库和应用程序，并生成一个 1.44MB 的磁盘镜像 `os.img`。

### 2. 运行仿真
使用 QEMU 运行：
```bash
make run
```

使用 Bochs 运行（用于验证分段保护，在app/main.c的第一个TODO中使用）：
```bash
make bochs
```

### 3. 清理构建
```bash
make clean
```

## 常见问题解答

1.bochs运行错误：

①`display library 'sdl2' not available`，解决方案，安装`sudo apt-get install bochs-sdl`

②`Message: ROM: couldn't open ROM image file '/usr/share/bochs/BIOS-bochs-latest'`或者`/usr/share/bochs/VGABIOS-lgpl-latest`，解决方案，安装`sudo apt-get install bochsbios`，`sudo apt-get install vgabios`。如果安装后还是不行（mac用户的安装目录可能和ubuntu虚拟机不一样），就把实验目录中的`bochsrc`文件中，`romimage`或者`vgaromimage`后面的`/usr/share/bochs`改成你自己的bochs安装目录，上面`xx-latest`可能会带有`.bin`的后缀名，可以自己根据情况增减。
