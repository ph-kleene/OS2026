# fork 系统调用实现说明 (Lab 3)

本文档列出了项目中实现 `fork` 系统调用所涉及的核心源文件及其详细功能解释。

## 1. 核心定义与配置

### `kernel/include/syscall.h`
- **角色**：系统调用号定义。
- **说明**：定义了 `SYS_FORK (3)`。这是用户态程序触发进程克隆功能的唯一识别码。

### `kernel/include/proc.h`
- **角色**：PCB 结构与函数声明。
- **说明**：
    - 声明了 `do_fork()` 内核函数。
    - 定义了 `MAX_PCB (5)`。由于项目采用静态槽位管理内存，`MAX_PCB` 限制了系统并发进程的总数，直接影响 `fork` 是否能成功分配新槽位。

## 2. 内核态实现逻辑

### `kernel/core/proc.c`
- **角色**：进程管理核心。
- **说明**：
    - **`do_fork()`**: 实现了传统的重量级进程克隆。
        1. **空闲槽位查找**：遍历 `pcb` 数组（从索引 1 开始，避开 Idle 进程），寻找状态为 `UNUSED` 的槽位。
        2. **物理内存拷贝**：将父进程所在的 128KB 物理内存（Slot）完整拷贝到子进程对应的槽位地址中。
        3. **上下文克隆**：拷贝父进程的内核栈（包含 `TrapFrame`），确保子进程能从相同的代码位置恢复执行。
        4. **返回值设置**：将子进程 `TrapFrame` 中的 `eax` 设为 0。这样当子进程被调度时，其从系统调用返回的值为 0，以此区分父子进程。
        5. **状态同步**：设置子进程的 PID、PPID，并将状态置为 `RUNNABLE` 等待调度。

### `kernel/core/irq_dispatch.c`
- **角色**：中断分发。
- **说明**：
    - 在 `syscallHandle` 中，当接收到 `SYS_FORK` 请求时，调用 `syscallFork` 包装函数。
    - `syscallFork` 执行 `do_fork()` 并将返回的子进程 PID（或错误码 -1）存入父进程的 `eax` 中，从而完成父进程的系统调用返回。

## 3. 用户态接口与测试

### `lib/syscall.c` & `lib/myos.h`
- **角色**：用户态库封装。
- **说明**：
    - `myos.h` 定义了 `int fork();` 供应用层调用。
    - `syscall.c` 实现具体的封装，通过汇编代码 `int $0x80` 配合 `SYS_FORK` 编号陷入内核。

### `app/main.c`
- **角色**：测试程序。
- **说明**：
    - 展示了 `fork` 的标准用法：通过判断返回值为 0 还是正数来执行不同的父子进程逻辑。
    - 包含了 **Fork 槽位上限测试**：演示了当进程数达到 `MAX_PCB` 限制时，`fork` 如何正确返回 `-1` 并处理失败情况。

## 4. 特殊引导

### `kernel/kernel.c`
- **角色**：内核初始化。
- **说明**：包含内核态内联汇编封装的 `idle_fork()`。该函数允许内核态进程通过中断方式直接克隆自身。
