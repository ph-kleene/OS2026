## 2. 进程与进程控制块 (PCB)
* [cite_start]**什么是进程**：进程是正在执行的程序实例 [cite: 85]。
* [cite_start]**PCB 的作用**：操作系统通过进程控制块 (PCB) 来管理每个进程 [cite: 86][cite_start]。PCB 包含了进程 ID、状态、寄存器上下文、内存信息等核心数据 [cite: 87]。
* **为什么需要 PCB**：
    * [cite_start]进程切换时需要保存完整的上下文 [cite: 89]。
    * [cite_start]需要跟踪进程状态（运行/就绪/阻塞） [cite: 90]。
    * [cite_start]需要知道进程的内存位置 [cite: 91]。

## 3. 进程创建与加载机制

### 3.1 `fork` 系统调用（进程创建）
* [cite_start]**作用**：创建一个新进程（子进程） [cite: 99]。
* [cite_start]**原理**：复制当前进程的全部内容 [cite: 100]。
* [cite_start]**返回值**：父进程返回子进程 PID，子进程返回 0 [cite: 101]。
* [cite_start]**内存语义**：传统 `fork` 会复制父进程的整个地址空间 [cite: 103][cite_start]。这样做的目的是保证父子进程独立运行，互不干扰 [cite: 104]。

### 3.2 `vfork` 系统调用（轻量级进程创建）
* [cite_start]**作用**：创建一个共享内存的子进程 [cite: 112]。
* [cite_start]**与 fork 的区别**：不复制内存，父子共享同一地址空间 [cite: 113]。
* [cite_start]**设计目的**：配合 `exec` 使用，避免不必要的数据复制 [cite: 114]。
* **存在的问题与解决方案**：
    * [cite_start]父子共享内存可能导致数据冲突 [cite: 116]。
    * [cite_start]**实验框架的限制**：限制同一时间只能有一个未 `exec` 的 `vfork` 子进程 [cite: 117]。
    * [cite_start]在执行 `exec` 时才会恢复独立的内存空间 [cite: 119]。

### 3.3 `exec` 系统调用（程序加载）
* [cite_start]**作用**：用新程序替换当前进程的内存空间 [cite: 128]。
* [cite_start]**参数**：扇区号和扇区数（从磁盘加载程序） [cite: 129]。
* [cite_start]**执行流程**：读取磁盘 $\rightarrow$ 覆盖当前内存 $\rightarrow$ 重置寄存器 [cite: 131]。
* [cite_start]**与 vfork 的配合**：`vfork` 创建的子进程初始共享父进程内存 [cite: 134][cite_start]。当调用 `exec` 时：恢复独立内存、加载新程序、减少父进程的 `vfork` 计数 [cite: 135]。

### 3.4 `exit` 与 `sleep`
* [cite_start]**exit（进程退出）**：结束当前进程，释放占用的槽位 [cite: 143][cite_start]。将状态设为 `UNUSED`，并调用调度器选择新进程 [cite: 144]。
* [cite_start]**sleep（阻塞式休眠）**：Lab2 中的 `sleep` 是忙等待（占用 CPU 循环检查） [cite: 146][cite_start]。Lab3 改进为：将状态设置为 `BLOCKED` 并设置 `sleep_ticks` [cite: 147]。

## 4. 进程调度与上下文切换

### 4.1 进程调度器
* [cite_start]**机制**：操作系统决定哪个进程获得 CPU 执行权 [cite: 155][cite_start]。通过时钟中断强制切换进程（抢占式调度） [cite: 156]。
* [cite_start]**调度算法**：本实验采用逆向扫描，即高 PID 优先 [cite: 158][cite_start]。因为高编号进程后创建，优先级应该更高 [cite: 159][cite_start]。Idle 进程 (PID 0) 作为保底进程 [cite: 160]。

### 4.2 上下文切换过程
* [cite_start]保存当前进程的寄存器到内核栈 [cite: 168]。
* [cite_start]更新当前进程状态（`RUNNING` $\rightarrow$ `RUNNABLE`） [cite: 169]。
* [cite_start]选择下一个进程 [cite: 171]。
* [cite_start]更新新进程状态（`RUNNABLE` $\rightarrow$ `RUNNING`） [cite: 172]。
* [cite_start]切换内核栈指针 (TSS) [cite: 173]。
* [cite_start]**切换内存基址 (GDT)**：实现进程隔离 [cite: 174][cite_start]。由于每个进程有不同的内存槽位，通过修改 GDT 中的基地址，可以让同一段虚拟地址映射到不同的物理空间 [cite: 176, 177]。

## 5. 内存管理：物理地址、逻辑地址与 GDT

### 5.1 物理内存布局
* [cite_start]本实验共划分了多个槽位，每个槽位 128KB，共计 512KB 用户空间 [cite: 185]。
    * [cite_start]**Slot 0** (0x00000 - 0x1FFF)：Idle进程 / PID 0 [cite: 184]。
    * [cite_start]**Slot 1** (0x20000 - 0x3FFF)：用户进程1 / PID 1 [cite: 184]。
    * [cite_start]**Slot 2** (0x40000 - 0x5FFF)：用户进程2 / PID 2 [cite: 184]。
    * [cite_start]**Slot 3** (0x60000 - 0x7FFF)：用户进程3 / PID 3 [cite: 184]。
    * [cite_start]**Slot 4** (0x80000 - 0x9FFFF)：用户进程4 / PID 4 [cite: 184]。
* [cite_start]通过 GDT 动态切换实现内存隔离 [cite: 186]。

### 5.2 地址转换与隔离实现
* [cite_start]**物理地址 vs 逻辑地址**：物理地址实际存在于内存芯片上 [cite: 196][cite_start]。逻辑地址是程序中使用的地址（从0开始） [cite: 197][cite_start]。段机制负责将逻辑地址转换为物理地址 [cite: 198]。
* [cite_start]在 Lab3 中，每个进程都认为自己从地址 0 开始运行 [cite: 200][cite_start]。但实际上被映射到了不同的物理槽位 [cite: 201][cite_start]。这种映射是通过 GDT 动态切换来实现的 [cite: 202]。
* [cite_start]**GDT（全局描述符表）**：包含基址 (Base)、界限 (Limit) 和特权级等属性 [cite: 209, 210][cite_start]。内核段基址为 0，特权级 DPL=0 [cite: 215][cite_start]。用户段基址动态变化，特权级 DPL=3 [cite: 216, 217]。
* [cite_start]**切换时机**：在 `schedule()` 函数中进行进程切换时，需调用 `update_user_seg_base()` [cite: 229][cite_start]。该函数修改 `gdt[SEG_UCODE]` 和 `gdt[SEG_UDATA]` 的基址字段 [cite: 230][cite_start]。切换后，进程访问的“地址 0”实际指向其对应槽位的物理地址 [cite: 231]。
