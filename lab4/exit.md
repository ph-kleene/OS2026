# exit 系统调用实现说明 (Lab 4)

本文档详细描述了 `exit` 系统调用在内核中的实现细节。

## 1. 核心定义与配置

### `kernel/include/syscall.h`
- **角色**：系统调用号定义。
- **说明**：定义了 `SYS_EXIT (6)`。进程通过该调用主动结束生命周期。

## 2. 内核态实现逻辑

### `kernel/core/proc.c`
- **角色**：进程退出管理。
- **说明**：
    - **`do_exit()`**:
        1. **状态变更**：将当前进程 `current->state` 置为 `ZOMBIE`。
        2. **孤儿进程处理**：遍历 `pcb` 数组，将所有父进程 PID 为当前 PID 的子进程的 `ppid` 重定向至 PID 0（Idle 进程）。这防止了子进程成为无主进程。
        3. **父进程通知/唤醒**：
            - 获取当前进程的父进程 PCB。
            - 检查父进程是否正处于 `WAIT_CHILD` 状态。
            - **活跃子进程计数**：检查父进程名下是否还有其他状态不为 `UNUSED` 且不为 `ZOMBIE` 的子进程。
            - **唤醒逻辑**：若当前进程是该父进程最后一个退出的子进程，则将父进程状态修改为 `RUNNABLE`，使其能够继续执行并清理僵尸进程。
        4. **vfork 并发控制处理**：如果当前进程通过 `vfork` 创建且正在共享父进程资源，则在退出时递减父进程的 `vfork_count`，释放并发限制。
        5. **调度**：执行 `schedule()`，由于当前进程已是 `ZOMBIE`，它将永远不会再被调度执行。

## 3. 资源回收

### 僵尸进程状态
- `exit` 调用后，进程的 PCB 槽位并未立即释放，而是保留 `ZOMBIE` 状态。
- 物理内存和 PCB 资源的最终释放由父进程在 `do_wait` 中完成。

## 4. 用户态接口

### `lib/syscall.c`
- **实现**：`void exit();`
- **汇编细节**：触发 `int $0x80` 后，由于内核不再调度该进程，用户态执行流在此处终结。

### `app/main.c`
- **应用场景**：子进程完成计算任务后必须显式调用 `exit()`，否则会继续向下执行父进程的代码，导致逻辑混乱。
