# schedule 进程调度实现说明 (Lab 4)

本文档详细描述了项目中基于时间片轮转（Round-Robin）的抢占式调度实现机制。

## 1. 调度算法与实现

### 时间片轮转 (Round-Robin)
从 Lab 4 开始，系统由简单的优先级/逆向扫描调度升级为公平的时间片轮转调度：

1.  **扫描策略**：`schedule()` 函数从当前进程的下一个 PID 开始循环扫描 `pcb` 数组（即 `(current->pid + 1) % MAX_PCB`）。
2.  **就绪判定**：寻找第一个处于 `RUNNABLE` 或 `RUNNING` 状态的进程。
3.  **Idle 进程低优先级**：调度器会优先搜索用户进程（PID 1 ~ MAX_PCB-1）。只有当系统中没有其他任何就绪的用户进程时，才会选择调度 Idle 进程（PID 0）。
4.  **时间片分配**：每当一个进程被选为 `next` 并投入运行时，其 `time_count` 会被初始化为 **5**（对应 50ms）。

## 2. 抢占机制与触发时机

### 时钟中断抢占
系统通过每 10ms 一次的硬件时钟中断（IRQ 0, 中断号 0x20）实现强制抢占：
*   在 `timerHandle` 中，当前运行进程的 `time_count` 递减。
*   当 `time_count` 减至 0 时，内核主动触发 `schedule()`。
*   这确保了没有任何一个计算密集型进程可以永久独占 CPU。

### 综合调度触发时机

| 触发位置 | 触发原因 | 行为描述 |
| :--- | :--- | :--- |
| **`timerHandle`** | **时间片耗尽/唤醒** | 进程时间片减为 0 时强制抢占；或者有休眠进程到期被唤醒。 |
| **`do_sleep`** | **主动阻塞** | 运行进程因调用 `sleep` 变为 `BLOCKED`，立即触发调度。 |
| **`do_sem_wait`** | **同步等待** | 信号量资源不足，进程回退 EIP 并阻塞，触发调度。 |
| **`do_wait`** | **子进程同步** | 父进程等待子进程退出，进入 `WAIT_CHILD` 状态，触发调度。 |
| **`do_exit`** | **任务终结** | 进程退出进入 `ZOMBIE` 状态，强制触发调度。 |
| **`do_fork` / `do_vfork`** | **新任务就绪** | 新进程创建完成，触发调度以平衡负载或让子进程尽早运行。 |

## 3. 进程切换底层细节

当 `schedule()` 确定了 `next` 进程后，底层的上下文切换由汇编代码与硬件特性共同完成：

### A. 寄存器现场与内核栈
*   **保存**：进入中断时，硬件压入 `SS, ESP, EFLAGS, CS, EIP`。`asmDoIrq` 汇编随后保存所有通用寄存器到旧进程的内核栈中。
*   **切换**：通过 `mov esp, [current]` 切换 CPU 的栈指针到新进程的上下文保存位置（即其 `TrapFrame` 所在位置）。
*   **恢复**：`asmDoIrq` 从新栈中 `pop` 恢复寄存器，最终通过 `iret` 回到新进程被切走时的指令位置。

### B. 地址空间隔离 (GDT 动态更新)
*   内核调用 `update_user_seg_base`。通过修改 GDT 中用户代码段和数据段的基地址，将新进程的虚拟地址映射到对应的物理槽位（128KB Slot），实现单 CPU 下的内存物理隔离。

### C. 特权级切换 (TSS)
*   更新 `tss.esp0` 为新进程内核栈的**栈底（即内核栈内存区域的最高地址）**。这保证了当该进程在用户态下次触发中断或系统调用时，CPU 能自动定位到其私有的内核栈起始位置，确保内核执行环境的独立性。
