# sem_post 系统调用实现说明 (Lab 4)

本文档详细描述了 `sem_post` 系统调用（V 操作）在内核中的实现细节。

## 1. 核心定义与配置

### `kernel/include/syscall.h`
- **角色**：系统调用号定义。
- **说明**：定义了 `SYS_SEM_POST (12)`。这是用户态程序请求释放信号量资源的唯一标识。

## 2. 内核态实现逻辑

### `kernel/core/proc.c`
- **角色**：同步原语实现。
- **说明**：
    - **`do_sem_post(int sem_id)`**:
        1. **参数校验**：检查 `sem_id` 是否在有效范围内且该信号量已在使用。若非法，返回 -1。
        2. **值更新**：无条件递增信号量计数值 `semaphores[sem_id].value`。
        3. **唤醒逻辑（FIFO）**：
            - 检查该信号量的 `wait_queue` 是否为空。
            - 若不为空，从链表头部移除第一个等待中的进程 PCB（记为 `woken`）。
            - 将 `woken->state` 修改为 `RUNNABLE`。
            - 将 `woken->next` 置为空，使其脱离等待队列。
        4. **调度策略说明**：为了优化高频同步场景下的系统性能，`do_sem_post` 在唤醒进程后**不主动**调用 `schedule()`。被唤醒的进程将进入 `RUNNABLE` 状态，并等待下一次时钟中断（每 10ms 触发一次）或当前进程主动放弃 CPU 时，由时间片轮转调度算法（Round-Robin）公平地选出执行。这种设计显著降低了上下文切换的频率，提升了系统吞吐量。

## 3. 原子性保证

### `kernel/x86/idt.c`
- **说明**：通过 **Interrupt Gate** 进入，整个 `do_sem_post` 过程在关中断状态下运行。这保证了在修改信号量值和操作 PCB 等待链表时，不会被其他进程或中断打断，从而避免了丢失唤醒（Lost Wakeup）或链表损坏。

## 4. 用户态接口

### `lib/myos.h`
- **声明**：`int sem_post(int sem_id);`

### `app/main.c`
- **应用场景**：在临界区代码执行完毕后调用，确保其他等待该信号量的进程能够进入临界区。
