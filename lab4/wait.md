# wait 系统调用实现说明 (Lab 4)

本文档详细描述了 `wait` 系统调用在内核中的实现细节。

## 1. 核心定义与配置

### `kernel/include/syscall.h`
- **角色**：系统调用号定义。
- **说明**：定义了 `SYS_WAIT (9)`。这是父进程请求同步等待并回收子进程资源的接口。

## 2. 内核态实现逻辑

### `kernel/core/proc.c`
- **角色**：进程回收管理。
- **说明**：
    - **`do_wait()`**:
        1. **子进程存在性检查**：扫描 `pcb` 数组，查找 `ppid` 等于当前进程 PID 且状态不为 `UNUSED` 的进程。若找不到任何子进程，立即返回 -1。
        2. **活跃子进程检查**：在找到子进程的基础上，检查是否存在状态**不是** `ZOMBIE` 的子进程。
        3. **阻塞逻辑（Mesa 重试）**：
            - 若存在活跃子进程，将当前进程状态设为 `WAIT_CHILD`。
            - **回退 EIP**：将 `current->tf->eip` 减去 2，使其在下次获得 CPU 时重新触发 `int $0x80` 进入 `do_wait`。
            - **调度**：调用 `schedule()` 切换到其他进程。
            - **返回**：返回 `SYS_WAIT` 以维持 `eax` 正确。
        4. **资源回收**：若所有子进程均已处于 `ZOMBIE` 状态：
            - 遍历并找到这些僵尸子进程，将其 PCB 状态设为 `UNUSED`。
            - 返回 0，代表成功回收子进程。

## 3. 阻塞与唤醒机制

### 唤醒源 (`do_exit`)
- 当子进程调用 `exit` 时，会检查父进程是否处于 `WAIT_CHILD` 状态。
- 如果当前退出进程是父进程的最后一个活跃子进程，它负责将父进程状态从 `WAIT_CHILD` 修改为 `RUNNABLE`。

### 自动关中断保护
- 整个 `do_wait` 的扫描、阻塞设置及调度过程都在关中断（中断门自动处理）状态下执行，确保了“检查-等待”操作的原子性，防止了父进程错过子进程退出的信号。

## 4. 用户态接口

### `lib/syscall.c`
- **实现**：`int wait();`

### `app/main.c`
- **说明**：父进程通过 `wait()` 确保在所有子进程结束并清理完毕后，再继续执行后续逻辑或打印测试结果。
